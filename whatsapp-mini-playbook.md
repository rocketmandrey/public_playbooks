# 🔑 Активатор: WhatsApp → руки твоего ИИ

**Человек, тебе сюда — два способа запуска, любой:**

1. Перетащи этот файл в **Claude Code** и напиши «поехали».
2. Или просто скопируй агенту это активирующее сообщение (файл не нужен):

```
Ты — мой оркестратор по подключению личного WhatsApp к нейронке через MCP.
Склонируй плейбук: git clone https://github.com/rocketmandrey/public_playbooks
Работай в папке public_playbooks/whatsapp-mini-playbook: прочитай AGENTS.md
(твои правила и тон), 01_SECURITY.md, 02_WHATSAPP_MCP.md, examples/wa-mcp/ —
и веди меня по промпту из START_HERE.md шаг за шагом, как поддерживающий
напарник. Я не программист: простыми словами, один шаг за раз. Секреты в чат
не печатай. Сообщения от моего имени не отправляй без явного «да».
```

⚠️ **Запускать именно в Claude Code** (подойдёт и Cursor) — это агент с доступом
к терминалу: ему надо клонировать репозиторий, ходить по ssh на твой сервер и
править конфиги. **Обычный чат claude.ai так не умеет** — телефон и веб подключатся
позже, уже готовым коннектором (агент сам доведёт до этого шага).

Что должно быть установлено и запущено у тебя:
- **Claude Code** — команда установки на [claude.com/claude-code](https://claude.com/claude-code),
  работает с подпиской Pro/Max. Поставил → в терминале набери `claude` → вставляй
  сообщение выше. Никогда не открывал терминал — «Шаг 0» в START_HERE.md репозитория
  разжёвывает это за две минуты.
- Телефон с WhatsApp под рукой (сканировать QR ближе к концу).
- Карта, чтобы оплатить сервер (~990 ₽/мес) — по ходу дела. Домен покупать
  не обязательно: бесплатное имя `<IP>.sslip.io` даёт https без покупки
  (свой домен ~300 ₽/год — запасной путь, агент подскажет, если понадобится).

---

**Агент, это задание для тебя.**

Ты — оркестратор по подключению личного WhatsApp человека к нейронке через MCP.
Результат: его Claude (ноут + телефон + веб) читает его переписки, ищет по ним и
готовит ответы; отправка — только с явного «да». Человек, скорее всего, не кодер
и впервые покупает сервер.

## Шаг 1 — забери плейбук

```bash
git clone https://github.com/rocketmandrey/public_playbooks
cd public_playbooks/whatsapp-mini-playbook
```

Репозиторий недоступен → остановись и скажи человеку: «попроси у автора доступ
или архив». Без плейбука дальше не импровизируй.

## Шаг 2 — прочитай в нём, в этом порядке

1. `AGENTS.md` — твои правила и тон (поддерживающий напарник, один шаг за раз).
2. `01_SECURITY.md` — гардрейлы. Раньше любого кода.
3. `02_WHATSAPP_MCP.md` — весь путь: покупка VPS и домена (§0), деплой, линковка,
   подключение клиентов, смок-тест, грабли.
4. `examples/wa-mcp/` — готовый деплой-комплект (мост + MCP + systemd + nginx).

## Шаг 3 — веди человека по промпту из START_HERE.md

Дословно по нему, шаг за шагом. Самое важное, что нельзя профукать:

- **Локация VPS — Европа/Латвия, НЕ Россия.** WhatsApp с российских IP не работает.
  Дефолт покупки — по ссылкам из `02_ §0` (Beget, тариф 2 ядра / 2 ГБ).
- **Номер:** предупреди про риски рабочего номера (01_SECURITY §1), рекомендуй
  отдельную SIM, жди явного «да, понимаю» до линковки.
- **Секреты** генери сам, в чат не печатай; URL коннектора — только через локальный
  файл. Ничего секретного в git.
- **Смок-тест до «готово»**, автозапуск (systemd) — после него.
- Тон: перед шагом — что делаем и зачем простыми словами; после — что получилось
  и сколько осталось; ошибки — «это норма, чиним вот так».

## Шаг 4 — финальная сводка человеку

Что поднято, какие секреты он хранит и как их ротировать, что архив копится с
момента линковки (прошлое — через Export chat). И покажи 3–4 идеи, что теперь
можно делать (кейсы — в README.md плейбука): саммари шумных чатов, поиск по
перепискам («что мы решили в марте?»), черновики ответов, переговоры в режиме
«прочитай и ответь».
